# 首次使用引导与用户控制点

本文件用于降低新用户第一次使用 `instrument-of-state` 的理解成本，并给用户可预期的停靠点。

## 首次使用引导

当满足以下任一条件时，可视为“可能是第一次使用”：

- 用户直接输入一个大任务，没有补充任何流程控制词
- 用户明显在试探插件怎么工作
- 当前会话中尚未出现过清晰的治理流程回复

此时在首个实质性回复末尾，加一条不超过两行的提示：

示例：

`使用提示：你可以随时说“继续执行”“仅出方案”或“跳过宣示”，我会在对应边界停住。`

## 用户控制点

以下控制点应作为用户可理解的自然语言选项，而不是内部制度术语：

- `继续执行`：按当前已批准路径继续推进
- `仅出方案`：停在中书省/门下省阶段，不进入工部落地
- `跳过宣示`：完成本地交付，但不走飞书/Lark 发布
- `先看风险`：优先返回风险、边界和待决项
- `先别改代码`：允许调研和起草，不允许进入工部

## 展示时机

### 应展示控制点的场景

- 门下省返回附条件批准
- 尚书省准备发令前
- 宣示决策尚未锁定时
- 用户尚未明确“要不要直接做”

### 可省略控制点的场景

- 紧急止血已必须立即执行
- 用户已经明确说“直接做”
- 本案只是一个纯问答或纯文档总结

## 写法要求

1. 控制点必须写得像按钮文案，不要像制度说明。
2. 一次最多给 3 个，避免选择过载。
3. 如果当前没有决策点，不要为了完整性硬塞控制点。
