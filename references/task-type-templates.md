# 任务类型输出模板

本文件为 `instrument-of-state` 提供四类常见任务的用户可见输出骨架。所有模板都遵循“先结论，后细节”的规则。

## 1. 故障 / 应急类

适用场景：

- 线上故障
- CI/CD 中断
- 热修复
- 回滚

优先强调：

1. 现在是否已止血
2. 是否还在影响用户
3. 下一步是谁处理什么

建议摘要写法：

- `当前阶段：兵部（应急/故障）牵头止血`
- `执行权限：可先止血，后补审`
- `下一动作：先恢复服务，再补门下审校与刑部验收`

## 2. 前端 / 设计类

适用场景：

- 页面改版
- 组件重做
- 视觉升级
- 交互优化

优先强调：

1. 当前是否还在方案阶段
2. 视觉方向是否已锁定
3. 是否进入真实实现

建议摘要写法：

- `当前阶段：中书省（起草方案）正在明确页面方向与交互目标`
- `执行权限：暂未进入工部（代码执行）`
- `下一动作：先确认视觉方向，再进入实现与验证`

## 3. 发布 / 宣示类

适用场景：

- 发布说明
- 飞书/Lark 文档
- 对外同步
- PR 仪式与交接

优先强调：

1. 本次是否需要正式宣示
2. 是现在发布、仅出文档，还是暂缓
3. 谁来发、发给谁

建议摘要写法：

- `当前阶段：礼部（文档/发布）正在准备正式文稿`
- `执行权限：可先产出文档，再决定是否通知收件人`
- `下一动作：确认收件范围与权限后进入宣示`

## 4. 文档 / 研究类

适用场景：

- 方案分析
- 代码调研
- ADR
- 文档整理

优先强调：

1. 当前是否只出方案，不动代码
2. 是否需要门下裁决
3. 用户是否还需要批准后续执行

建议摘要写法：

- `当前阶段：中书省（起草方案）正在整理结论与边界`
- `执行权限：当前仅产出方案，不直接落地代码`
- `下一动作：门下省审校后，再决定是否转入执行`

## 使用规则

1. Shangshu 应尽早判断任务类型。
2. 选定任务类型后，优先采用对应摘要骨架。
3. 即使采用模板，也不能跳过朝政进度、门下裁决卡和最终行动。
